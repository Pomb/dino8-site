CUSTOM SHADERS

Learn how to hook up GLSL shaders in Dino8 in 5 steps.

STEP 1 Load a shader

Dino8 lets you write custom fragment shaders in GLSL. A shader is a small program that runs on the GPU and transforms every pixel on screen.

Load a shader with shader_load(name, source). The source can be a file path or inline GLSL:

Two ways to load:
shader_load("fx", "shaders/fx.glsl") file path (contains "/" or ends in ".glsl")
shader_load("fx", [[ ...glsl... ]]) inline source (everything else)

We'll use the collapse shader as our example. It snaps pixel colors toward a target color — any pixel close enough to the target gets pulled all the way to it, creating a posterization effect.

Use these macros instead of raw GLSL for cross-platform compatibility:

VARYING declares an input from the vertex stage
TEXTURE2D(tex, uv) samples a texture
FRAGCOLOR the output pixel color
shaders/collapse.glsl
VARYING vec2 UV;
uniform sampler2D SCREEN;
uniform float amount;
uniform vec3 target;
void main() {
    vec4 color = TEXTURE2D(SCREEN, UV);
    float dist = distance(color.rgb, target);
    float threshold = amount * 1.8;
    float snap = step(dist, threshold);
    FRAGCOLOR = vec4(
        mix(color.rgb, target, snap),
        color.a
    );
}
main.d8
-- load from file:
-- local collapse = shader_load("collapse", "shaders/collapse.glsl")

-- or load inline:
local collapse = shader_load("collapse", [[
    VARYING vec2 UV;
    uniform sampler2D SCREEN;
    uniform float amount;
    uniform vec3 target;
    void main() {
        vec4 color = TEXTURE2D(SCREEN, UV);
        float dist = distance(color.rgb, target);
        float threshold = amount * 1.8;
        float snap = step(dist, threshold);
        FRAGCOLOR = vec4(
            mix(color.rgb, target, snap),
            color.a
        );
    }
]])

function _draw()
  cls(1)
  for i = 1, 10 do
    rectf(10, 10 + i * 10, 118, 16 + i * 10, (i % 15) + 1)
  end
end
STEP 2 Initial uniforms

Our collapse shader declares two uniforms — amount and target — but we haven't set them yet, so the effect does nothing.

Pass a table of initial values as the third argument to shader_load:

Value types:
amount = 0.5 number → float
target = rgb(16) table of 3 → vec3
tint = {1,0,0,1} table of 4 → vec4

The rgb(index) function returns a palette color as {r, g, b} with values from 0.0 to 1.0 — exactly what shaders expect.

With amount at 0.5 and target set to palette color 16 (white), any pixel whose color is close enough to white will snap to white.

To apply a shader as a full-screen post-process effect, pass true as the last argument to shader_load. You can also add or remove shaders from the post-process pipeline later with post_process_add(handle) and post_process_remove(handle).

ENGINE UNIFORMS

These are set automatically — just declare them in your GLSL:

SCREEN          sampler2D   current screen pixels
RESOLUTION      vec2        screen size
TIME            float       seconds since start
MOUSE           vec2        normalized mouse pos
PALETTE         sampler2D   palette as 1D texture
PALETTE_SIZE    float       number of palette colors
main.d8
local collapse = shader_load("collapse", [[
    VARYING vec2 UV;
    uniform sampler2D SCREEN;
    uniform float amount;
    uniform vec3 target;
    void main() {
        vec4 color = TEXTURE2D(SCREEN, UV);
        float dist = distance(color.rgb, target);
        float threshold = amount * 1.8;
        float snap = step(dist, threshold);
        FRAGCOLOR = vec4(
            mix(color.rgb, target, snap),
            color.a
        );
    }
]], {
  amount = 0.5,
  target = rgb(16),
}, true)
STEP 3 Update at runtime

Use shader_set to change uniform values every frame. This is how you animate or control shader effects interactively.

New functions:
shader_set(handle, name, value) set a uniform — pass a number or a table

For a float uniform, pass a number. For a vec3, pass a table of 3 values — rgb() returns exactly that, so you can feed it directly.

Here we use up/down to control the collapse amount, and A/B to cycle through palette colors as the target. Try it — at high amounts, most of the screen snaps to a single color.

main.d8
local amount = 0.0
local target_idx = 16

local collapse = shader_load("collapse", [[
    VARYING vec2 UV;
    uniform sampler2D SCREEN;
    uniform float amount;
    uniform vec3 target;
    void main() {
        vec4 color = TEXTURE2D(SCREEN, UV);
        float dist = distance(color.rgb, target);
        float threshold = amount * 1.8;
        float snap = step(dist, threshold);
        FRAGCOLOR = vec4(
            mix(color.rgb, target, snap),
            color.a
        );
    }
]], {
  amount = 0.0,
  target = rgb(16),
}, true)

function _update()
  if btn(2) then
    amount = min(amount + 0.02, 1.0)
  end
  if btn(3) then
    amount = max(amount - 0.02, 0.0)
  end
  shader_set(collapse, "amount", amount)

  if btnp(4) then
    target_idx = (target_idx % 16) + 1
    shader_set(collapse, "target", rgb(target_idx))
  end
end

function _draw()
  cls(1)
  for i = 1, 10 do
    rectf(10, 10 + i * 10, 118, 16 + i * 10, (i % 15) + 1)
  end
  for i = 1, 5 do
    circf(20 + i * 20, 64 + sin(T + i) * 20, 8, (i % 15) + 1)
  end
  print("collapse: " .. string.format("%.2f", amount), 4, 4, 7)
end
STEP 4 Localized shaders

So far the shader affects the entire screen. To apply it to only part of the screen, use shader_apply with a mask.

New functions:
shader_apply(handle, mask_rt, x, y) apply shader mid-draw using a render target as mask
rt_create(slot, w, h) create a render target
rt_target(slot) redirect drawing to a render target (or back to screen)

A mask is a render target you draw into. The mask controls where the shader runs:

transparent pixels → no effect (screen unchanged)
any color → shader applies here

First, create a render target with rt_create(slot, w, h). The slot is a number from 1 to 8 — it's just an ID to refer to this render target later. Then redirect drawing into it with rt_target(slot), draw your shapes, and call rt_target() with no argument to switch back to the screen.

Build up the mask with as many draw calls as you like — circles, rectangles, lines, whatever. Anywhere you draw is where the shader will run. Here we combine a circle and a rectangle into a single mask:

The mask can be any size and positioned anywhere on screen with the x, y arguments to shader_apply.

Note that we use shader_apply inside _draw() instead of the post-process flag — this runs the shader on the current screen contents at that moment, rather than as a final post-process pass.

main.d8
local amount = 0.0

local collapse = shader_load("collapse", [[
    VARYING vec2 UV;
    uniform sampler2D SCREEN;
    uniform float amount;
    uniform vec3 target;
    void main() {
        vec4 color = TEXTURE2D(SCREEN, UV);
        float dist = distance(color.rgb, target);
        float threshold = amount * 1.8;
        float snap = step(dist, threshold);
        FRAGCOLOR = vec4(
            mix(color.rgb, target, snap),
            color.a
        );
    }
]], {
  amount = 0.0,
  target = rgb(16),
})

-- create mask in render target slot 1 (slots 1-8)
local mask = 1
rt_create(mask, 80, 80)
rt_target(mask)
circf(40, 30, 25, 7)
rectf(15, 50, 65, 70, 7)
rt_target()  -- back to screen

function _update()
  if btn(2) then
    amount = min(amount + 0.02, 1.0)
  end
  if btn(3) then
    amount = max(amount - 0.02, 0.0)
  end
  shader_set(collapse, "amount", amount)
end

function _draw()
  cls(1)
  for i = 1, 10 do
    rectf(10, 10 + i * 10, 118, 16 + i * 10, (i % 15) + 1)
  end

  -- only the masked area gets collapsed
  shader_apply(collapse, mask, 24, 24)

  print("collapse: " .. string.format("%.2f", amount), 4, 4, 7)
end
STEP 5 Multi-color masks

The mask isn't just on/off — the shader receives it as a texture (MASK), so your GLSL can read the mask's colors to make per-pixel decisions.

Here we extend the collapse shader with two targets. The mask uses two colors:

off-white area → collapses toward target_a
red area → collapses toward target_b

The shader picks between targets by comparing the mask pixel's red channel. To sample the mask correctly, convert screen UV to mask UV with the engine-provided MASK_OFFSET and MASK_SIZE uniforms.

Any area left transparent in the mask is still unaffected — so you can combine spatial masking with color-driven logic in a single pass.

main.d8
local amount = 0.0

local collapse = shader_load("collapse", [[
    VARYING vec2 UV;
    uniform sampler2D SCREEN;
    uniform sampler2D MASK;
    uniform vec2 MASK_OFFSET;
    uniform vec2 MASK_SIZE;
    uniform float amount;
    uniform vec3 target_a;
    uniform vec3 target_b;

    void main() {
        vec4 color = TEXTURE2D(SCREEN, UV);
        vec2 mask_uv = (UV - MASK_OFFSET) / MASK_SIZE;
        vec4 m = TEXTURE2D(MASK, mask_uv);

        // off-white circle -> target A, red background -> target B
        vec3 target = m.g > 0.5
            ? target_a
            : target_b;

        float dist = distance(color.rgb, target);
        float threshold = amount * 1.8;
        float snap = step(dist, threshold);
        FRAGCOLOR = vec4(
            mix(color.rgb, target, snap),
            color.a
        );
    }
]], {
  amount = 0.0,
  target_a = rgb(11),  -- off-white
  target_b = rgb(7),   -- red
})

-- mask: off-white circle over red background
local mask = 1
rt_create(mask, 128, 128)
rt_target(mask)
rectf(0, 0, 128, 128, 7)
circf(64, 64, 30, 11)
rt_target()

function _update()
  if btn(2) then
    amount = min(amount + 0.02, 1.0)
  end
  if btn(3) then
    amount = max(amount - 0.02, 0.0)
  end
  shader_set(collapse, "amount", amount)
end

function _draw()
  cls(1)
  for i = 1, 10 do
    rectf(10, 10 + i * 10, 118, 16 + i * 10, (i % 15) + 1)
  end
  for i = 1, 5 do
    circf(20 + i * 20, 64 + sin(T + i) * 20, 8, (i % 15) + 1)
  end

  -- apply shader only where mask is drawn
  shader_apply(collapse, mask)

  print("collapse: " .. string.format("%.2f", amount), 4, 4, 7)
end

WHAT NEXT

You now know the core shader workflow: load, configure uniforms, update at runtime, and apply with masks. From here you can:

  • Write shaders that use TIME for animation
  • Use PALETTE to look up colors by index in GLSL
  • Animate the mask offset with the x, y args to shader_apply
  • Stack multiple post-process shaders with post_process_add for layered effects

Check the shader API reference for the full function list.
